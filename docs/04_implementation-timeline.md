## 技術選定（推奨案）

### AI サービス

| 候補 | 無料枠 | 特徴 | 推奨度 |
|------|--------|------|--------|
| **Gemini API（Google）** | 無料枠あり（60 RPM） | マルチモーダル対応（画像+テキスト解析可能） | ★★★ 最推奨 |
| Groq（Llama 3） | 無料枠あり | 超高速レスポンス、テキストのみ | ★★☆ |
| Cloudflare Workers AI | 無料枠あり | エッジ実行、Llama系モデル | ★☆☆ |

> **推奨**: Gemini API — 画像解析も可能なため MEMORIA の写真→感情解析に最適

### DB / Auth / Storage

| 要素 | 推奨 | 理由 |
|------|------|------|
| **DB** | Supabase（PostgreSQL） | 無料枠で十分、リアルタイム対応、Row Level Security |
| **Auth** | Supabase Auth（Google OAuth） | DB と一体管理、Next.js SDK あり |
| **Storage** | Supabase Storage | 同一プラットフォームで完結、画像の公開URL発行が容易 |

### DB スキーマ概要

```sql
-- ユーザー
users (
  id UUID PK,           -- Supabase Auth uid
  nickname TEXT,
  avatar_state JSONB,    -- アバター状態
  created_at TIMESTAMPTZ
)

-- 思い出（日記）
memories (
  id UUID PK,
  user_id UUID FK → users,
  photo_url TEXT,
  input_text TEXT,        -- ユーザー入力テキスト
  diary_text TEXT,        -- AI生成日記
  emotion JSONB,          -- {type: "cat", score: 0.85, tags: ["自由", "好奇心"]}
  animal_type TEXT,       -- cat / rabbit / lion / bear / fox
  created_at TIMESTAMPTZ
)

-- 夢・目標
dreams (
  id UUID PK,
  user_id UUID FK → users,
  title TEXT,             -- 短冊に書いた夢
  deadline DATE,
  created_at TIMESTAMPTZ
)

-- ロードマップステップ
steps (
  id UUID PK,
  dream_id UUID FK → dreams,
  order_index INT,
  title TEXT,
  description TEXT,
  is_completed BOOLEAN DEFAULT false,
  completed_at TIMESTAMPTZ
)
```

## リスク & 対策

| リスク | 影響度 | 対策 |
|--------|--------|------|
| AI API のレート制限 | 高 | レスポンスキャッシュ / デモ用固定レスポンスを用意 |
| 画像解析の精度が低い | 中 | テキスト入力を必須化し、テキストメインで解析 |
| 3名の並行開発でコンフリクト | 中 | 画面単位で担当を明確に分離 / feature ブランチ運用 |
| デプロイ時の環境変数漏れ | 高 | Vercel の Environment Variables で管理、`.env.local` を `.gitignore` |
| 時間不足で Should が全滅 | 低 | Must を Phase 3 までに死守、Should は Phase 4 のバッファ |
